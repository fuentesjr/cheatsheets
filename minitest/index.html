Install & Setup

Add to Gemfile (bundled with Ruby stdlib, but pin explicitly for control)

gem "minitest"

# then
bundle install

Install directly

gem install minitest

Minimal test file — require "minitest/autorun" registers an at_exit runner

require "minitest/autorun"

class TestMath < Minitest::Test
  def test_addition
    assert_equal 4, 2 + 2
  end
end

Run a single file

ruby -Ilib -Itest test/test_math.rb

Spec-style file (same power, BDD syntax)

require "minitest/autorun"

describe "Math" do
  it "adds numbers" do
    _(2 + 2).must_equal 4
  end
end
Running Tests (CLI options)

Flags pass to the runner after the file, or via TESTOPTS with Rake. Set SEED or use --seed to reproduce order.

OptionEffect
-h, --helpDisplay runner options
-s, --seed NSet random seed for test order
-v, --verboseVerbose output; show each test name & time
-n, --name PATRun tests matching string or /regexp/
-e, --exclude PATSkip tests matching string or /regexp/
-p, --prideRainbow "pride" output 🌈

Run one test by name

ruby -Itest test/test_math.rb -n test_addition
ruby -Itest test/test_math.rb -n "/add/"

Reproduce a failing random order

ruby -Itest test/test_math.rb --seed 12345
# or
SEED=12345 ruby -Itest test/test_math.rb

Via Rake (default test task from Rake::TestTask)

rake test
rake test TESTOPTS="--seed 12345 -v"
rake test TEST=test/test_math.rb

Rakefile test task

require "rake/testtask"

Rake::TestTask.new(:test) do |t|
  t.libs << "test"
  t.libs << "lib"
  t.test_files = FileList["test/**/test_*.rb"]
  t.warning = true
end
Assertions

All take an optional final msg. Every assertion has a refute_* negation with identical signature.

AssertionPasses when
assert(test, msg=nil)test is truthy
assert_equal(exp, act)exp == act
assert_same(exp, act)exp.equal?(act) (same object)
assert_nil(obj)obj.nil?
assert_empty(obj)obj.empty?
assert_includes(coll, obj)coll.include?(obj)
assert_match(re, str)re =~ str
assert_instance_of(cls, obj)obj.instance_of?(cls)
assert_kind_of(cls, obj)obj.kind_of?(cls)
assert_respond_to(obj, m)obj.respond_to?(m)
assert_predicate(obj, m)obj.send(m) is truthy (e.g. :empty?)
assert_operator(a, op, b)a.send(op, b) is truthy
assert_in_delta(exp, act, d=0.001)|exp-act| <= d
assert_in_epsilon(a, b, e=0.001)relative error within epsilon
assert_raises(*classes) { }block raises one of the classes; returns the exception
assert_throws(sym) { }block throws sym
assert_output(out, err) { }stdout/stderr match String or Regexp (nil = ignore)
assert_silent { }block produces no output
assert_path_exists(path)File.exist?(path)

Capture & inspect a raised exception

err = assert_raises(ArgumentError) do
  Integer("nope")
end
assert_match(/invalid value/, err.message)

Assert against output

assert_output("hello\n", "") { puts "hello" }
assert_output(/\d+ rows/, nil) { report }

flunk / pass / skip — control flow inside a test

flunk "should not reach here"      # always fails
pass                                # counts as one assertion
skip "pending API" unless online?  # marked S, not F
Spec Expectations (BDD)

Wrap the subject with _(obj), value(obj), or expect(obj). Each must_* has a wont_* negation.

Expectation≡ Assertion
_(x).must_equal yassert_equal
_(x).must_be_nilassert_nil
_(x).must_be_emptyassert_empty
_(x).must_include yassert_includes
_(x).must_match reassert_match
_(x).must_be_instance_of Cassert_instance_of
_(x).must_be_kind_of Cassert_kind_of
_(x).must_respond_to massert_respond_to
_(x).must_be :>, 0assert_operator
_(x).must_be_close_to y, dassert_in_delta
_(x).must_be_within_epsilon yassert_in_epsilon
_ { ... }.must_raise Eassert_raises
_ { ... }.must_output outassert_output
_ { ... }.must_throw sassert_throws

Block-form expectations use _ { }

describe "parse" do
  it "rejects bad input" do
    _ { parse("x") }.must_raise ParseError
  end
end

Disable expectations globally to avoid Object monkey-patching

MT_NO_EXPECTATIONS=1 ruby -Itest test/spec.rb
Test Structure & Lifecycle

setup / teardown run around each test method

class TestUser < Minitest::Test
  def setup
    @user = User.new(name: "Ada")
  end

  def teardown
    @user = nil
  end

  def test_name
    assert_equal "Ada", @user.name
  end
end

Spec hooks: before / after (per example), plus setup/teardown

describe User do
  before { @user = User.new }
  after  { @user = nil }

  it("has a name") { _(@user.name).must_be_nil }
end

Force declaration order instead of random

class TestSeq < Minitest::Test
  i_suck_and_my_tests_are_order_dependent!
  # or override:
  def self.test_order; :alpha; end   # :random (default), :sorted, :alpha
end

Shared behavior via a module included into a Test class

module Serializable
  def test_round_trips
    assert_equal @obj, load(dump(@obj))
  end
end

class TestJSON < Minitest::Test
  include Serializable
  def setup; @obj = {a: 1}; end
end
Mocks & Stubs

Minitest::Mock — set expectations, then verify they were met

mock = Minitest::Mock.new
mock.expect(:save, true)
mock.expect(:name, "Ada")

assert mock.save
assert_equal "Ada", mock.name
mock.verify   # raises MockExpectationError if unmet

Expect with specific arguments

mock.expect(:charge, :ok, [100, "usd"])
mock.charge(100, "usd")     # ok
# mock.charge(50, "usd")    # MockExpectationError

Validate args with a block (and keyword args)

mock.expect(:send_mail, true) do |to:, subject:|
  to.end_with?("@example.com")
end
mock.send_mail(to: "a@example.com", subject: "Hi")

stub — temporarily replace a method for the block's duration

Time.stub(:now, Time.at(0)) do
  assert_equal 0, Time.now.to_i
end

# stub with a callable / dynamic value
user.stub(:admin?, -> { true }) do
  assert dashboard.visible?
end

Pass a Mock as the stub value to also assert the call

m = Minitest::Mock.new
m.expect(:call, 42, [1, 2])
obj.stub(:compute, m) do
  assert_equal 42, obj.compute(1, 2)
end
m.verify

stub requires require "minitest/autorun" (or minitest/mock) and is available inside Test/spec bodies.

Benchmarking

Assert an algorithm fits a complexity curve — subclass Minitest::Benchmark

require "minitest/benchmark"

class BenchSort < Minitest::Benchmark
  def bench_linear
    assert_performance_linear 0.99 do |n|
      Array.new(n) { rand }.sort
    end
  end
end
MethodAsserts
assert_performance_constant(threshold=0.99)O(1)
assert_performance_linear(threshold=0.99)O(n)
assert_performance_powerO(n^k)
assert_performance_logarithmicO(log n)
self.bench_rangeOverride to set input sizes (default 1,10,100,1k,10k)

Minitest::Reporters.use! [
  Minitest::Reporters::SpecReporter.new

class TestX < Minitest::Test
  focus
  def test_only_this; assert true; end
end

Gotchas & Tips

Argument order is (expected, actual) — reversing them produces confusing diff output

assert_equal 4, result   # exp, act  ✔
assert_equal result, 4   # backwards ✘

Wrap spec subjects — bare obj.must_equal triggers a deprecation; use _()

_(user.name).must_equal "Ada"   # ✔
user.name.must_equal "Ada"      # deprecated global expectation

Test method names must start with test_ in Minitest::Test subclasses, or they never run

assert_nil vs assert_equal nil, x — the latter warns; prefer the dedicated assertion

Random order is intentional. If tests fail only sometimes, you have shared state — don't reach for i_suck_..., fix the leak

A test with no assertions is reported but "passes" — Minitest doesn't fail empty tests; use pass only if intentional

Custom assertion: just wrap assert/refute in a method available to your tests

module Minitest::Assertions
  def assert_even(n, msg = nil)
    msg ||= "Expected #{n.inspect} to be even"
    assert n.even?, msg
  end
end
